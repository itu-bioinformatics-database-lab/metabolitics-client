MetaboliticsDB - About the project

Creating an account
Head to the top menu and click on ‘Login’ section, you will see Signup text under the login button, click it and you will be redirected to the signup page to create a new account.



The user will have to fill the following form, then click the ‘Sign Up’ button, then the user will get a confirmation message for a successful registration, and can now login and proceed to use the website as a registered user.



Search

Search For Pathways/Metabolites/Reactions

  • In metabolitesDB we can search for the following from the main page:
    • Pathways
    • Metabolites
    • Reactions
You can enter the first few characters and our autocomplete service will show you the names close to your search.



Search Metabolomics Analysis


You can find this subsection in the top menu Search Section. For better user experience our search services support autocomplete for possible search texts based on the search characters you enter. In metabolitesDB we support Searching public studies based on two search criteria:
  • Search studies based on Metabolites
    • Enter the metabolite name you wish to find studies that include it, then you will be provided with a list of studies that include it.
  • Search studies based on Pathways
    • Enter Pathway name, Change, Diff quantifier, and Diff Amount and then press the Add button and click search to get the results, you may choose multiple pathways to search at once.

Supported data formats
in metabliticsdb we provided Sample files for each format we support, they are provided in the documentation section. Metabolites Names should be from one of our supported Datasets:
  • CheBI
  • Hmdb
  • Kegg
  • PubChem

You can access the sample file formats from the Top menu Documentation Section.




CSV Format:

First column entries should contain metabolites names and the second column entries should include their corresponding values/measurements.



JSON Format:

The Keys should be metabolites names and the values should be their corresponding values/measurements.



Metabolitics Format:

In our format we have two sheets:
  • Sheet one:
    • Sheet name should be “data”, it includes subjects ids/names and their corresponding metabolites names and Values/measurements.
    • The first row (starting from the second column)should include subjects ids/names.
    • The first column(starting from the second row)should include metabolite names from one of our supported datasets, each metabolite row entry should include the corresponding values/measurements for the subjects.


  • Sheet two:
    • Sheet name should be “meta”, it should include study name, group label, subjects ids/names, and their corresponding labels.
    • The first row should include ‘study name’ as first Entry and the Study name of your choice as the second entry.
    • The second row should include ‘control/healthy/wildtype group’ in the first entry and its corresponding label in the second entry.
    • In the third row, the first entry should be ‘subject id’ and the second entry should be ‘group’ which will represent the subject label.
    • For the next columns(starting from row 4), column 1 should have the subjects ids/names and column 2 should have the corresponding labels of the subject.


Metabolomics Workbench:

Please refer to the following link for a detailed tutorial on Metabolimics Workbench MWtab files format: https://www.metabolomicsworkbench.org/data/tutorials.php

Analyze
In the analyze section, we provided 3 options for the user to use our analyze service:
  • Uploading Studies files to be analyzed.
  • Manually Add Metabolites Changes to be Analyzed.
  • Use a Sample Metabolite Changes List.

Upload File

when choosing this option you will be directed to the following page with various file formats that MetaboliticsDB support, for detailed information about the formats refer to Supported data formats section.

Choose your preferred format and upload your study file, and you will be redirected to the following page:


Now, whether you choose to Upload File, Enter Metabolites Manually or sample metabolites list you will be directed to this page with your study data processed to be sent to be analyzed, but first, you have to provide the necessary information that maybe not be included in your uploaded file:
  • If you are not a registered user, we will be sending the analysis result to your email address so you should fill the email entry.
  • All users should fill the Disease or Condition entry for the study.
  • The user will have to provide us with a Study name.
  • The user will have the option to make his study public or private (private studies are only provided for registered users).
  • We also give the option to manually add or remove metabolites changes.
  • Last Step, the user has to choose the analysis algorithm, we currently support two analysis algorithms:
    • Metabolitics (our own analysis)
    • Direct Pathway Mapping
  • After clicking the Analyze button, the user will be directed to the result page or sent to the main page if it’s not ready immediately, but the users can check there study analysis result at any time from Browse analysis section, for private studies the user will have to be logged in to view his private studies.
Accessing Analysis Results

Registered Users

Registered Users will be able to send their data to be analyzed and Control whether the analysis results will be publicly available or kept private and only shown for the user.

To access analysis results for registered users, Head to top menu to Account section and choose Past Analysis subsection and you will be redirected to your Private studies results list, then you check the results or compare between them, you can check public studies results from the same page if you choose to by clicking ‘Public’ button.

Unregistered Users

Unregistered users are able to send their data to be analyzed but they will need to provide us with their Email, once the analysis results are ready they will be notified by email with the link to access it, an important notice is that those studies done by unregistered users will be publicly available. To access publicly available analysis results, head to the top menu and select the ‘Browse’ section and click ‘Browse Analysis’ and you will be redirected to analysis results page.

Analysis results page

when we access an analysis result page, The analysis results arepresented in multiple levels of details in both visual and tabular forms a bar plot for the top 20 pathways with the highest absolute diff values will is provided.

Below the bar plot, in a tabular form, all the pathways are listed with their computed diff values.Next to each pathway, two buttons are placed. The left button visualizes the corresponding pathway where each node represents a metabolite, and each edge represents a reaction. Edges are colored and thickened according tothe corresponding reactions’ diff value. This form of visualization allows the user to quickly inspect what parts of thepathway experience the most change.


The right button, on the other hand,lists all the pathway reactions and their diff values in tabular form.



Comparison of different analysis results for different datasets
MetaboliticsDB allows users to compare and contrast their metabolomics data analysis results to their previous analysis results, other users’publicly available analysis results, and diseases and physiologicalconditions stored in the database. to achieve that you have to access the browse analysis subsection from the Browse section in the top menu.

Next the user will have to choose a study and press it, it will expand and show the subjects inside that study, in each subject tab there is a checkbutton, in order to compare between two subjects the user will have to click the check buttons for all the subjects he wishes to compare.


Next the user will go up to the top menu and click the button ‘Compare ’, in our case we choose two subjects from the same study, Next we will be redirected to the comparison page. The comparison page features a heatmap at the top where the rows represent pathways with the highest variance in terms of their diff values among the compared analysis results, and the columns represent the compared analysis results.Each cell in the heatmap is colored according to the corresponding pathway diff values. in the lower part of the page we will find each subjects pathways values.



Disease Association
Disease association
  • Another novel feature of the MetaboliticsDB is that for each metabolomics analysis result, the closest set of diseases and physiological conditions are computed based on the similarity of the pathway-level metabolic behavior.
  • MetaboliticsDB stores metabolomics analysis results of a number of diseases (pre-computed and recorded in its database). Then, for each user submitted analysis run, the user is presented the top 5 diseases (e.g., diabetes) that have most similarity to the currently analyzed metabolomics data in terms of the activity level distribution over the metabolic pathways.
To check disease association, we access our analysis results page (refer to Accessing Analysis Section in the Tutorial), then we will notice under the pathways bar plot there is a Disease Section, in it we will provide the top 5 similar cases and their similarity value to the analysis.
